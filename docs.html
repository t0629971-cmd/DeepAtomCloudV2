01 Getting Started

DeepAtom runs as a standalone Python application on Windows. There's nothing to compile — install the dependencies and launch the GUI.

Prerequisites

  • Windows 7, 8, 10 or 11 (x64)
  • Python 3.8 or newer
  • 2 GB RAM minimum (4 GB recommended)

Quick install

pip install -r requirements.txt
python DeepAtom.py
Tip: Enable Proactive Defense by installing wmi, watchdog, psutil and pywin32 — see the Download page for the full step-by-step guide.

First scan

Open DeepAtom, choose Quick Scan for a fast sweep of common infection points, or drag and drop a file/folder directly onto the window for an on-demand check.

02 User Manual

The DeepAtom interface is organized around three scan modes and three protection toggles.

Scan modes

  • Quick Scan — startup folders, running processes, temp directories
  • Full Scan — every file on selected drives
  • Custom Scan — a specific file, folder, or list of paths

Protection toggles

  • Real-Time Guard — background monitoring of new processes
  • Proactive Defense — behavioral analysis for ransomware, injection, tampering, and persistence
  • System32 Guard — extra write-protection for critical system folders

Quarantine & history

Every detection is moved to an encrypted quarantine folder rather than deleted outright, so you can restore a false positive with one click. The History tab keeps a searchable log of every scan DeepAtom has run.

03 Architecture Guide

DeepAtom is organized into four cooperating layers:

1
PreventiveSHA256 blacklist + YARA signatures + Real-Time Guard
2
DetectiveML classification + behavioral monitoring
3
AnalyticalSandbox execution of unknown binaries
4
ResponsiveQuarantine + optional cloud threat sharing

Component stack

GUI (PyQt5)
    ↓
Scanner Core (SHA256 + YARA + ML)
    ↓
Proactive Defense (WMI + Filesystem + Registry)
    ↓
Behavioral Monitor + Sandbox

The scanner core and the behavioral monitor run independently — disabling one does not disable the other. This is what lets Proactive Defense catch fileless-style attacks that never touch disk long enough for a signature-based scan to see them.

04 Training the ML Model

DeepAtom's classifier is a LightGBM RandomForest model trained on a labeled dataset of PE files.

Feature set

Each file is reduced to 15 static features before it reaches the model — a deliberately compact set chosen for speed (~50ms per prediction) without sacrificing accuracy on the primary target: Windows PE malware.

  • Header and section-level structural metadata
  • Import/export table characteristics
  • Entropy and packing indicators
  • Resource and overlay attributes

Retraining on your own data

# dataset.csv: labeled samples with the 15 extracted features
python train_model.py --dataset dataset.csv --output model.pkl
Note: Retraining requires a balanced dataset of clean and malicious samples. Poorly labeled data will directly hurt detection accuracy.

05 API Reference

DeepAtom exposes a small local API surface for scripting and automation.

Scan a file

from deepatom.scanner import Scanner

scanner = Scanner()
result = scanner.scan_file("C:/path/to/file.exe")

print(result.verdict)     # "clean" | "malicious" | "suspicious"
print(result.confidence)  # 0.0 – 1.0
print(result.layer)       # which layer produced the verdict

Cloud hash lookup

GET https://deepatomserver.onrender.com/api/lookup?sha256=<hash>

Returns known classification data for a given SHA256 hash. Only hashes are sent — never file contents — and this call can be disabled entirely in Settings → Cloud.

Response shape

{
  "sha256": "…",
  "known": true,
  "verdict": "malicious",
  "family": "ransomware",
  "first_seen": "2026-05-11"
}
